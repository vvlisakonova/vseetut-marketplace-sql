# Разработка витрины данных и решение ad hoc задач — маркетплейс «ВсёТут»

Разработка витрины данных и решение ad hoc задач на основе данных маркетплейса «ВсёТут». Включает формирование агрегированной витрины поведения пользователей и четыре аналитических SQL-запроса.

---

## Описание

Цель проекта: подготовить витрину данных маркетплейса и на её основе ответить на аналитические вопросы заказчика.

В ходе работы:
- разработана витрина `product_user_features` с агрегированными характеристиками поведения пользователей
- витрина ограничена тремя регионами с наибольшим количеством заказов и включает только доставленные и отменённые заказы
- решены четыре ad hoc задачи: сегментация пользователей, ранжирование по среднему чеку, статистика по регионам, анализ активности по первому месяцу заказа

## Данные

Проект выполнен на базе данных маркетплейса `ds_ecom`, которая включает следующие таблицы:

| Таблица | Описание |
|---------|----------|
| `ds_ecom.orders` | Заказы пользователей |
| `ds_ecom.users` | Информация о пользователях |
| `ds_ecom.order_payments` | Платежи по заказам |
| `ds_ecom.order_reviews` | Отзывы и рейтинги заказов |
| `ds_ecom.order_items` | Товары в заказах |
| `ds_ecom.product_user_features` | Готовая витрина данных |

## Структура проекта

### Часть 1. Витрина данных `product_user_features`

Витрина формируется запросом с фильтрами:
- только заказы со статусом «Доставлено» или «Отменено»
- только три региона с наибольшим количеством заказов (Москва, Санкт-Петербург, Новосибирская область)
- каждая строка — уникальная пара «клиент — регион»

Витрина включает:
- базовую информацию о клиенте: `user_id`, `region`, `first_order_ts`, `last_order_ts`, `lifetime`
- информацию о заказах: `total_orders`, `avg_order_rating`, `num_orders_with_rating`, `num_canceled_orders`, `canceled_orders_ratio`
- информацию о платежах: `total_order_costs`, `avg_order_cost`, `num_installment_orders`, `num_orders_with_promo`
- бинарные признаки: `used_money_transfer`, `used_installments`, `used_cancel`

---

**Задача 2. Ранжирование пользователей с 3+ заказами по среднему чеку**

Топ-15 пользователей с наибольшим средним чеком среди тех, кто совершил 3 и более заказов. Наибольший средний чек — ~14 716, наименьший в топ-15 — ~5 526. Большинство пользователей из топ-15 (12 из 15) сделали ровно 3 заказа.

**Задача 4. Активность пользователей по первому месяцу заказа в 2023 году**

- Наибольшее число новых пользователей — в ноябре (4 703), наименьшее — в январе (465).
- Средняя оценка заказов стабильна: 4.1–4.3 во все месяцы.
- Средняя продолжительность активности наибольшая у «январских» пользователей (12 дней), наименьшая — у «декабрьских» (2 дня).
- Доля использующих денежный перевод: 19–22% в зависимости от месяца.
